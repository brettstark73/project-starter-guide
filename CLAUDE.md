### Development Workflow Preferences

#### **Quality Automation Integration**
- **Always test template functionality** before committing
- **Run smoke tests** on all templates after changes
- **Validate documentation accuracy** against actual implementation
- **Security scanning** for configuration exposure and hardcoded secrets

#### **Release Process**
- **Free tier changes**: Direct to GitHub with proper versioning
- **Paid tier features**: Develop in private repository, selective open-sourcing
- **Quality automation**: Continue as separate open-source project (lead generation)
- **Documentation**: Always update both free and paid tier docs

#### **Monetization Guidelines**
- **Never compromise free tier quality** - it's the lead magnet
- **Paid features should be additive** - not restrictive of free features
- **Quality automation remains open source** - builds credibility and adoption
- **Template marketplace approach** - free samples, paid premium collections
